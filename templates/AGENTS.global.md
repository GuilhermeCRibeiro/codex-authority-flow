# Codex Global Authority

Aplique a skill `codex-authority-flow` no início de toda tarefa que analise, modifique, teste, revise ou publique código.

Não peça ao usuário para escolher modelo, esforço, skill, metodologia ou quantidade de agentes.

Regras globais:

- classifique a tarefa como simples, normal ou complexa;
- use Superpowers como metodologia principal;
- use apenas as skills necessárias;
- não combine metodologias completas concorrentes;
- use OpenSpec apenas para mudanças relevantes;
- use Graphify apenas em análise estrutural complexa;
- use UI UX Pro Max apenas em trabalho visual ou de experiência;
- use Fable Judge somente como verificador independente;
- use Self-Learning somente após validação;
- não permita edição concorrente dos mesmos arquivos por subagentes;
- não declare sucesso sem executar validações aplicáveis;
- preserve escopo, segurança e padrões do repositório.

O `AGENTS.md` específico do projeto pode adicionar stack, arquitetura, comandos e restrições mais detalhadas.
